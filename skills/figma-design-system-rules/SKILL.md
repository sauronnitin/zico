# Figma → Design System Rules Skill

## ⚠️ PRIVACY GATE — ALWAYS RUN FIRST

Before accessing ANY Figma data, you MUST ask:

> "I'm about to read design data from your Figma file via the Figma MCP.
> This data stays between you, Claude Code, and Figma's own servers (no third parties).
>
> - **File to be read:** [state the file name or URL if known]
> - **Data accessed:** component names, design tokens (colors, typography, spacing), screenshots, layer structure
> - **NOT accessed:** Figma account credentials, billing info, other workspaces
>
> ✅ Proceed? Or would you like to use a different file?"

Only proceed after explicit user confirmation.

---

## What This Skill Does

Generates project-specific design system rules that teach Claude (and Cursor) exactly
how to translate Figma designs into your codebase — consistently, every time.

Output file depends on target agent:
- **Claude Code** → `CLAUDE.md` (appended as a `## Figma Design System Rules` section)
- **Cursor** → `.cursor/rules/figma-design-system.mdc`
- **Codex CLI** → `AGENTS.md`

## Workflow (Run After Privacy Gate Confirmation)

### Step 1 — Gather Design Context
```
Use Figma MCP tools in this order:
1. get_design_context    → structured design data (tokens, components, styles)
2. get_screenshot        → visual reference of the target frame/component
3. get_metadata          → file/page structure (if get_design_context is truncated)
4. get_variable_defs     → design tokens (colors, spacing, typography variables)
```

### Step 2 — Analyze Codebase
Examine the project for:
- Component file locations and naming conventions
- Styling approach (Tailwind classes, CSS Modules, styled-components, etc.)
- Design token definitions (CSS variables, JS theme objects, Tailwind config)
- Import patterns and barrel files
- Existing similar components to match conventions

### Step 3 — Generate Rules
Write rules that are **specific and actionable**, not generic. For each rule category:

**Component Discovery**
- Where components live: `src/components/`, `app/components/`, etc.
- Naming: PascalCase, kebab-case files, index.ts barrels
- Re-use existing components rather than creating duplicates

**Design Token Usage**
- Never hardcode hex colors or pixel values
- Reference tokens: `text-primary` not `#1a1a2e`, `spacing-4` not `16px`
- Map Figma token names to codebase token names

**Styling Conventions**
- Tailwind: use existing `cn()` utility, respect `tailwind.config.ts` breakpoints
- CSS Modules: mirror Figma layer names as class names
- Styled-components: use theme variables, not raw values

**Figma MCP Integration Workflow**
- Always run `get_screenshot` before implementation for visual validation
- Run `get_design_context` for structured spacing/typography values
- Validate final output against screenshot before declaring done

**Asset Handling**
- Export SVGs from Figma to `public/icons/` or `src/assets/`
- Images: use `next/image` (Next.js) or `<img loading="lazy">` (standard)
- Never inline Figma CDN URLs directly in code

### Step 4 — Save Rules
Append to the target file under a clearly marked section:
```
## Figma Design System Rules
<!-- Generated by figma-design-system-rules skill — [date] -->
[generated rules here]
```

### Step 5 — Validate
Implement one simple component using the rules and visually compare against the
Figma screenshot to confirm accuracy before proceeding.

## Data Flow & Privacy Notes

```
Claude Code (local)
      ↓
Figma MCP Server (local process on your machine)
      ↓
Figma API (api.figma.com) — Figma's own servers
      ↓
Design data returned to Claude Code (local)
```

- ❌ No data goes to Anthropic's training
- ❌ No data goes to any third-party services
- ✅ Figma's own privacy policy applies (same as using the Figma app)
- ⚠️ claude-mem will NOT store Figma MCP responses (excluded from memory capture)

---

## Prerequisites

1. Figma MCP server must be installed and running:
   ```bash
   npx @figma/mcp-server@latest
   ```
2. MCP configured in `~/.claude/claude_desktop_config.json` or via Claude Code MCP settings
3. Active Figma account with access to the target file
